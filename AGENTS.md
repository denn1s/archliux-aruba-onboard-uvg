# Context for AI Agents - UVG Aruba Onboard (Universal Guide)

## Overview

You are helping a Linux user connect to **UVG CAMPUS CENTRAL** WiFi network using HPE Aruba Onboard enrollment system.

**Critical Facts**:
- **Network**: UVG CAMPUS CENTRAL (802.1X WPA-Enterprise, certificate-based auth)
- **Requirement**: This solution requires NetworkManager + wpa_supplicant
- **Compatibility**: `iwd` is not supported (its TLS stack fails with Aruba RADIUS servers)
- **Enrollment**: Browser-based via Aruba CloudGuest portal
- **Platform**: Any Linux distro that supports NetworkManager

---

## Understanding the Components

### What's in the .deb Package

The `Aruba_Onboard_Installer.deb` contains three binaries:

```
/usr/share/aruba-onboard/bin/
├── onboard-srv    # D-Bus service for NetworkManager integration
├── onboard-ui     # Qt5 GUI for enrollment
└── onboard-cli    # Command-line tool (rarely used)
```

**How enrollment works**:

1. **User clicks `arubadp://` link** from university portal
2. **onboard-ui launches** (via xdg protocol handler)
3. **onboard-ui connects to Aruba CloudGuest** via HTTPS
4. **User accepts certificates** in the GUI
5. **CloudGuest generates certificates** (CA, client cert, PKCS#12 key)
6. **onboard-ui downloads certificates** to `~/.aruba-onboard/`
7. **onboard-ui calls onboard-srv** via D-Bus RPC
8. **onboard-srv creates NetworkManager connection** with all credentials
9. **Connection is ready** - user can connect immediately

**The problem**: Steps 7-8 often fail on non-Ubuntu systems, leaving you with certificates but no NetworkManager connection configured.

## Certificate Storage

After enrollment, certificates are stored in:

```bash
~/.aruba-onboard/
├── UVG CAMPUS CENTRAL-ca.pem       # CA certificate (~895 bytes)
├── UVG CAMPUS CENTRAL-user.pem     # Your device certificate (~1261 bytes)
├── UVG CAMPUS CENTRAL-key.p12      # PKCS#12 encrypted private key (~2276 bytes)
└── profiles.adpdata                 # Enrollment metadata (JSON)
```

### Certificate Details

**Inspect the certificates**:
```bash
# View your device certificate
openssl x509 -in ~/.aruba-onboard/*-user.pem -noout -text

# Key information:
openssl x509 -in ~/.aruba-onboard/*-user.pem -noout -subject -dates -issuer

# Expected output:
# subject: CN=username@uvg.edu.gt, O=52e107d4ed9b11ec8b7e52e2ed9a47ed
# issuer: CN=aruba-cloudauth-ecdsa-ca-intermediate, O=Aruba CloudAuth
# notAfter: <date 1 year from enrollment>
```

**Extract identity from certificate**:
```bash
CERT_FILE=$(ls ~/.aruba-onboard/*-user.pem)
SCOPE_ID=$(openssl x509 -in "$CERT_FILE" -noout -subject | grep -oP 'O\s*=\s*\K[^,]+')
IDENTITY="anonymous@${SCOPE_ID}.cloudauth.net"

echo "Your 802.1X identity: $IDENTITY"
# Should be: anonymous@52e107d4ed9b11ec8b7e52e2ed9a47ed.cloudauth.net
```

**The PKCS#12 Password Problem**:
- The private key (`*-key.p12`) is encrypted with a random 32-character password
- This password is generated by Aruba CloudGuest during enrollment
- onboard-srv receives it via D-Bus and stores it in NetworkManager
- **You cannot decrypt the PKCS#12 without this password**
- **The password is NOT derivable** - it must be captured during enrollment

## Manual NetworkManager Configuration

If onboard-srv fails to create the connection automatically, you can do it manually.

### Option 1: With PKCS#12 Password (Ideal)

If you captured the password from D-Bus or logs:

```bash
# Set variables
CA_CERT=$(ls ~/.aruba-onboard/*-ca.pem)
CLIENT_CERT=$(ls ~/.aruba-onboard/*-user.pem)
PRIVATE_KEY=$(ls ~/.aruba-onboard/*-key.p12)
IDENTITY="anonymous@52e107d4ed9b11ec8b7e52e2ed9a47ed.cloudauth.net"
PASSWORD="<captured-password-here>"

# Create connection
nmcli connection add \
    type wifi \
    con-name "UVG CAMPUS CENTRAL" \
    ifname wlan0 \
    ssid "UVG CAMPUS CENTRAL" \
    wifi-sec.key-mgmt wpa-eap \
    802-1x.eap tls \
    802-1x.identity "$IDENTITY" \
    802-1x.ca-cert "$CA_CERT" \
    802-1x.client-cert "$CLIENT_CERT" \
    802-1x.private-key "$PRIVATE_KEY" \
    802-1x.private-key-password "$PASSWORD" \
    802-1x.phase1-auth-flags 1 \
    connection.autoconnect no

# Connect
nmcli connection up "UVG CAMPUS CENTRAL"
```

### Option 2: Decrypt PKCS#12 to Unencrypted PEM (If You Have Password)

If you have the password but want to avoid storing it in NetworkManager:

```bash
# Extract unencrypted private key
PKCS12_FILE=$(ls ~/.aruba-onboard/*-key.p12)
openssl pkcs12 -in "$PKCS12_FILE" -nocerts -nodes -out ~/.aruba-onboard/private-key.pem

# Use unencrypted key in connection
nmcli connection add \
    type wifi \
    con-name "UVG CAMPUS CENTRAL" \
    ifname wlan0 \
    ssid "UVG CAMPUS CENTRAL" \
    wifi-sec.key-mgmt wpa-eap \
    802-1x.eap tls \
    802-1x.identity "anonymous@52e107d4ed9b11ec8b7e52e2ed9a47ed.cloudauth.net" \
    802-1x.ca-cert "$(ls ~/.aruba-onboard/*-ca.pem)" \
    802-1x.client-cert "$(ls ~/.aruba-onboard/*-user.pem)" \
    802-1x.private-key "$HOME/.aruba-onboard/private-key.pem" \
    802-1x.phase1-auth-flags 1 \
    connection.autoconnect no
```

**Security note**: Unencrypted private key is less secure. Set proper permissions:
```bash
chmod 600 ~/.aruba-onboard/private-key.pem
```

### Option 3: Re-Enroll to Capture Password (If Lost)

If you don't have the password:

```bash
# Backup old enrollment
mv ~/.aruba-onboard ~/.aruba-onboard.backup

# Run enrollment with D-Bus monitoring to capture password
dbus-monitor --session "sender='net.aruba.onboard'" 2>&1 | tee /tmp/dbus-capture.log &
MONITOR_PID=$!

# Run enrollment (use your actual enrollment URL)
/path/to/onboard-ui "arubadp://uswest4.cloudguest.central.arubanetworks.com/..."

# After enrollment completes, stop monitoring
kill $MONITOR_PID

# Search for password in logs
grep -i "private-key-password" /tmp/dbus-capture.log
# or
grep -oP 'string "[a-zA-Z0-9]{32}"' /tmp/dbus-capture.log | tail -1
```

## Network Requirements

### Must Have (Any Distro)

1. **NetworkManager** (not iwd)
   ```bash
   # Check if running:
   systemctl is-active NetworkManager
   # Should show: active

   # Install if missing:
   # Debian/Ubuntu: sudo apt install network-manager
   # Fedora:        sudo dnf install NetworkManager
   # Arch:          sudo pacman -S networkmanager
   # openSUSE:      sudo zypper install NetworkManager
   # Gentoo:        emerge --ask net-misc/networkmanager
   ```

2. **wpa_supplicant** (for EAP-TLS)
   ```bash
   # Usually installed with NetworkManager
   # Verify it's running:
   ps aux | grep wpa_supplicant
   ```

3. **Qt5 libraries** (for onboard-ui GUI)
   ```bash
   # Debian/Ubuntu: sudo apt install libqt5core5a libqt5gui5 libqt5widgets5 libqt5svg5
   # Fedora:        sudo dnf install qt5-qtbase qt5-qtsvg
   # Arch:          sudo pacman -S qt5-base qt5-svg
   # openSUSE:      sudo zypper install libQt5Core5 libQt5Gui5 libQt5Svg5
   ```

### Backend Requirements

**iwd must NOT be running**:
```bash
# Check status
systemctl is-active iwd
# Should show: inactive (or failed/not-found)

# If active, disable it:
sudo systemctl stop iwd
sudo systemctl disable iwd
sudo systemctl restart NetworkManager
```

**Why iwd fails**:
- iwd has its own TLS implementation
- It's incompatible with Aruba ClearPass RADIUS servers
- Fails with: `EAP completed with eapFail`, `4-Way handshake failed, reason: 23`
- wpa_supplicant uses OpenSSL for TLS, which works correctly

## Enrollment Process (Step-by-Step)

### 1. Prerequisites

- Internet connection (phone hotspot, home WiFi, etc.)
- NetworkManager installed and running
- iwd disabled
- Qt5 libraries installed

### 2. Get Enrollment URL

Visit the UVG enrollment portal:
```
https://uswest4.cloudguest.central.arubanetworks.com/portal/scope.cust-52e107d4ed9b11ec8b7e52e2ed9a47ed/cda-user-portal/passpoint
```

Click:
1. "Yes, I already have HPE Aruba Networking Onboard"
2. Ignore "unsupported OS" warnings
3. "Install using HPE Aruba Networking Onboard app"

### 3. Extract Enrollment Link (If App Doesn't Launch)

If clicking the button doesn't launch the app:

1. Open browser console (F12 → Console tab)
2. Click the button again
3. Copy the `arubadp://` link from console
4. Launch manually:
   ```bash
   /path/to/onboard-ui "arubadp://uswest4.cloudguest.central.arubanetworks.com/provision?data=..."
   ```

### 4. Run Enrollment

**Simple approach** (if onboard-srv works):
```bash
# Install and run the GUI
/usr/share/aruba-onboard/bin/onboard-ui "arubadp://..."
```

**Advanced approach** (with password capture):
```bash
# Start onboard-srv in background
/usr/share/aruba-onboard/bin/onboard-srv &
SRV_PID=$!

# Monitor D-Bus for password
dbus-monitor --session "sender='net.aruba.onboard'" 2>&1 | tee /tmp/enrollment-dbus.log &
MONITOR_PID=$!

# Run enrollment UI
/usr/share/aruba-onboard/bin/onboard-ui "arubadp://..."

# After enrollment completes:
kill $MONITOR_PID
kill $SRV_PID

# Extract password from logs
grep -oP '"private-key-password".*?string "(\K[^"]+)' /tmp/enrollment-dbus.log
```

### 5. Verify Certificates Downloaded

```bash
ls -lh ~/.aruba-onboard/
# Should have: *-ca.pem, *-user.pem, *-key.p12

# Check certificate validity
openssl x509 -in ~/.aruba-onboard/*-user.pem -noout -dates
# Should show notAfter: 1 year from now
```

### 6. Create NetworkManager Connection

If onboard-srv didn't create it automatically, use the manual steps from "Manual NetworkManager Configuration" above.

### 7. Connect

```bash
nmcli connection up "UVG CAMPUS CENTRAL"

# Verify connection
nmcli device status
ping 8.8.8.8
```

## Debugging

### Issue 1: WiFi Interface Doesn't Exist

**Symptom**: `nmcli device status` doesn't show wlan0

**Diagnosis**:
```bash
# Check if hardware exists
ls /sys/class/ieee80211/
# Should show phy0 (or similar)

# Check if interface exists
ip link show wlan0
# Fails? Interface not created
```

**Fix**:
```bash
# Manually create interface
sudo iw phy phy0 interface add wlan0 type station
sudo ip link set wlan0 up

# Restart NetworkManager
sudo systemctl restart NetworkManager

# Verify
nmcli device status | grep wlan0
```

**Permanent fix** (systemd service):
```bash
# Create /etc/systemd/system/wifi-interface.service
sudo tee /etc/systemd/system/wifi-interface.service > /dev/null << 'EOF'
[Unit]
Description=Create WiFi Network Interface
After=sys-subsystem-net-devices-phy0.device
Wants=sys-subsystem-net-devices-phy0.device

[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/usr/sbin/iw phy phy0 interface add wlan0 type station
ExecStart=/usr/sbin/ip link set wlan0 up

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now wifi-interface
```

### Issue 2: EAP Authentication Fails

**Symptom**: Connection fails with authentication errors

**Diagnosis**:
```bash
# Live logs during connection attempt
journalctl -u NetworkManager -f &
LOGGER_PID=$!

nmcli connection up "UVG CAMPUS CENTRAL"

kill $LOGGER_PID
```

**Common errors**:

1. **"eapFail" or "reason: 23"** → iwd is running (disable it)
2. **"certificate verification failed"** → Wrong CA cert or expired
3. **"private key password incorrect"** → Wrong PKCS#12 password
4. **"TLS alert"** → Check phase1-auth-flags setting

**Verify connection parameters**:
```bash
nmcli connection show "UVG CAMPUS CENTRAL" | grep 802-1x

# Expected:
# 802-1x.eap:                    tls
# 802-1x.identity:               anonymous@52e107d4ed9b11ec8b7e52e2ed9a47ed.cloudauth.net
# 802-1x.ca-cert:                /home/USER/.aruba-onboard/UVG CAMPUS CENTRAL-ca.pem
# 802-1x.client-cert:            /home/USER/.aruba-onboard/UVG CAMPUS CENTRAL-user.pem
# 802-1x.private-key:            /home/USER/.aruba-onboard/UVG CAMPUS CENTRAL-key.p12
# 802-1x.private-key-password:   <hidden>
# 802-1x.phase1-auth-flags:      1
```

### Issue 3: Enrollment Completes but No Connection Created

**Symptom**: Certificates downloaded but NetworkManager connection missing

**Cause**: onboard-srv failed to create connection

**Solution**: Create connection manually (see "Manual NetworkManager Configuration" section above)

### Issue 4: Protocol Handler Doesn't Work

**Symptom**: Clicking `arubadp://` link does nothing

**Fix protocol handler registration**:
```bash
# Create desktop file
sudo tee /usr/share/applications/aruba-onboard.desktop > /dev/null << 'EOF'
[Desktop Entry]
Type=Application
Name=Aruba Onboard
Exec=/opt/aruba-onboard/bin/onboard-ui %u
MimeType=x-scheme-handler/arubadp
NoDisplay=true
EOF

# Register handler
xdg-mime default aruba-onboard.desktop x-scheme-handler/arubadp
update-desktop-database

# Verify
xdg-mime query default x-scheme-handler/arubadp
# Should show: aruba-onboard.desktop
```

## Testing Without Disconnecting

If user is on phone hotspot for internet access, testing UVG connection will disconnect them.

**Safe testing script**:
```bash
#!/bin/bash
# Ask user for hotspot credentials first!
HOTSPOT_NAME="User's Hotspot"
HOTSPOT_PASS="password"

LOG="/tmp/uvg_test_$(date +%s).log"
exec 1> >(tee -a "$LOG")
exec 2>&1

echo "=== UVG Connection Test $(date) ==="

# Try connection (with timeout)
timeout 30 nmcli connection up "UVG CAMPUS CENTRAL"
RESULT=$?

echo "Connection result: $RESULT"

# Test internet if connected
if [ $RESULT -eq 0 ]; then
    echo "Testing internet..."
    timeout 10 ping -c 3 8.8.8.8
fi

# Capture logs
echo "=== NetworkManager Logs ==="
journalctl -u NetworkManager --since "2 minutes ago" -n 50

# Disconnect
nmcli connection down "UVG CAMPUS CENTRAL"

# Reconnect to hotspot
sleep 2
nmcli device wifi connect "$HOTSPOT_NAME" password "$HOTSPOT_PASS"

echo "=== Test Complete ==="
echo "Log: $LOG"
```

## Technical Architecture

### Enrollment Flow Diagram

```
User's Browser
  │
  ├─> Aruba CloudGuest Portal (HTTPS)
  │   └─> Generates enrollment token
  │
  └─> Clicks arubadp:// link
      │
      ▼
  onboard-ui (Qt5 GUI)
      │
      ├─> Connects to CloudGuest via HTTPS
      ├─> Validates TLS certificate
      ├─> Sends enrollment token
      │
      ▼
  CloudGuest Server
      │
      ├─> Generates certificates (CA, client, PKCS#12)
      ├─> Sends to onboard-ui
      │
      ▼
  onboard-ui saves to ~/.aruba-onboard/
      │
      ├─> Calls onboard-srv via D-Bus RPC
      │   └─> add_network_connection(ssid, certs, password, ...)
      │
      ▼
  onboard-srv (D-Bus service)
      │
      ├─> Creates NetworkManager connection
      ├─> Sets all 802.1x parameters
      └─> Stores PKCS#12 password in keyring
```

### D-Bus Communication

onboard-ui → onboard-srv communication happens via D-Bus:

```
Service: net.aruba.onboard
Interface: net.aruba.onboard
Method: add_network_connection

Parameters:
  - ssid: "UVG CAMPUS CENTRAL"
  - ca-cert: "/home/user/.aruba-onboard/UVG CAMPUS CENTRAL-ca.pem"
  - client-cert: "/home/user/.aruba-onboard/UVG CAMPUS CENTRAL-user.pem"
  - private-key: "/home/user/.aruba-onboard/UVG CAMPUS CENTRAL-key.p12"
  - private-key-password: "<random-32-chars>"
  - identity: "anonymous@52e107d4ed9b11ec8b7e52e2ed9a47ed.cloudauth.net"
  - eap-method: "tls"
```

To capture this communication:
```bash
dbus-monitor --session "sender='net.aruba.onboard'"
```

## Security Considerations

### Certificate Validity

- **Lifetime**: 1 year from enrollment
- **Device-specific**: Tied to `/etc/machine-id` (re-enrollment needed per device)
- **Cannot be shared**: Each device needs separate enrollment

### PKCS#12 Password

- **Random 32-character alphanumeric string**
- **Generated by CloudGuest server** (not derivable)
- **Must be captured during enrollment** (cannot be recovered later)
- **Stored in NetworkManager keyring** (encrypted at rest)

### Listening Services

onboard-srv listens on `localhost:43671` when running:
- **No authentication** (any local process can connect)
- **Only accepts localhost connections** (not exposed to network)
- **Recommendation**: Only run during enrollment, not as always-on service
- See security analysis in `docs/PROFESSOR-GUIDE.md`

## Quick Diagnostic Script

```bash
#!/bin/bash
echo "=== UVG WiFi Diagnostic ==="
echo ""

echo "1. NetworkManager Status:"
echo -n "   NetworkManager: "; systemctl is-active NetworkManager 2>&1
echo -n "   iwd: "; systemctl is-active iwd 2>&1
echo -n "   wpa_supplicant: "; pgrep -a wpa_supplicant > /dev/null && echo "running" || echo "not running"
echo ""

echo "2. WiFi Interface:"
echo -n "   Hardware (phy0): "; [ -d /sys/class/ieee80211/phy0 ] && echo "exists" || echo "missing"
echo -n "   Interface (wlan0): "; ip link show wlan0 >/dev/null 2>&1 && echo "exists" || echo "missing"
nmcli device status | grep -E "DEVICE|wlan"
echo ""

echo "3. Certificates:"
if [ -d ~/.aruba-onboard/ ]; then
    ls -lh ~/.aruba-onboard/*.pem ~/.aruba-onboard/*.p12 2>/dev/null
    echo "   Certificate expiry:"
    openssl x509 -in ~/.aruba-onboard/*-user.pem -noout -dates 2>/dev/null | grep notAfter
else
    echo "   Not enrolled (no ~/.aruba-onboard/)"
fi
echo ""

echo "4. NetworkManager Connection:"
nmcli connection show | grep -E "NAME|UVG CAMPUS CENTRAL"
echo ""

echo "5. Network Visibility:"
nmcli device wifi list | grep -E "SSID|UVG CAMPUS CENTRAL" | head -5
echo ""

echo "=== End Diagnostic ==="
```

Save as `/tmp/diagnostic.sh`, run with `bash /tmp/diagnostic.sh`.
